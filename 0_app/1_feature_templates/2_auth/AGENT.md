# Auth Feature Template Rules

- Use this folder only as an auth feature template.
- Do not implement real product behavior in this template folder.
- When a real project needs auth, copy and adapt this template into `0_app/1_features/2_auth/` after user confirmation.
- Keep template proposal files together:
  - `spec.md`
  - `acceptance.md`
  - `feature.manifest.yaml`
  - `AGENT.md`
- Keep route ownership examples declared in `feature.manifest.yaml` so real features can adapt them.
